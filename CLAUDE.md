# Project Governance — bpetite

Deterministic byte-level BPE tokenizer. Pure Python + `regex`. Python 3.12, `uv`-managed, CI on ubuntu-latest and macos-latest.

The PRD (`bpetite-prd-v2.md`) is the source of truth for all product requirements. This file governs agent behavior only.

## Protocol Zero

All code in this repository is owned by Dinesh. No exceptions.

- Do not add "generated by AI", "written by Claude", "AI-assisted", or similar attribution in code, comments, docstrings, documentation, commit messages, or PR descriptions.
- Do not use self-referential AI language in any repository artifact.
- Do not add co-authored-by trailers or AI attribution headers/footers.
- If uncertain whether something counts as AI attribution, leave it out.

## Git Workflow

- Never commit without asking Dinesh first.
- Never open a PR without asking Dinesh first.
- Never push directly to `main`. All changes go through PRs.
- Branch naming: `feat/short-description`, `fix/short-description`, `chore/short-description`.
- Commit format: conventional commits — `feat:`, `fix:`, `chore:`, `docs:`, `refactor:`, `test:`.
- Before marking a PR as ready: all four quality gates must pass.
- Do not merge without explicit approval from Dinesh.

### Commit Mechanism

Direct `git commit` calls are prohibited. The repo-local `require-commitall-approval` pre-commit hook blocks any commit that did not go through the approval workflow.

To commit:

- Run `/commitall` (or `bash scripts/commitall.sh`).
- Review the diff, write a commit message, and explicitly approve at the prompt.
- No commit proceeds without explicit approval.

Never:

- Run `git commit` directly.
- Create or delete `.git/COMMIT_APPROVED` manually.
- Attempt to skip or bypass the pre-commit hook.

## Quality Gate

Every change must pass all four commands before PR readiness:

```bash
uv run pytest
uv run ruff check .
uv run ruff format --check .
uv run mypy --strict
```

Do not skip any. Do not treat a subset as sufficient.

## Python Standards

- 4-space indentation. 88-character line length (ruff default).
- `snake_case` for functions, methods, variables, modules. `PascalCase` for classes. `UPPER_SNAKE_CASE` for constants.
- Single leading underscore for internal/private members (`_trainer.py`, `_internal_method`).
- Type hints on all public and internal function signatures — parameters and return types. `mypy --strict` enforces this.
- Use lowercase generic syntax (`list[int]`, `dict[str, Any]`) — no `typing.List` or `typing.Dict`.
- Imports: stdlib → third-party → local, separated by blank lines. Absolute imports only.
- No bare `except:`. Catch specific exceptions. Never silently swallow exceptions.
- Use custom exception classes for domain-specific errors with precise, grep-friendly messages.
- Google-style docstrings on all public classes and methods. Module-level docstrings on every module.
- No emojis anywhere — code, comments, docstrings, commit messages, PR descriptions, CLI output. Zero tolerance.
- No implicit coercions between `bytes`, `bytearray`, and `memoryview` — strict byte typing.
- Ruff is the linter and formatter. Not Black, not autopep8, not yapf.

### Code Comments

- Comments exist to explain _why_, never to restate _what_ the code does.
- No banners, section dividers, ASCII art, or decorative comment blocks.
- No TODO without a concrete description of what remains and why it was deferred.
- No commented-out code. Dead code is deleted, not preserved behind `#`.
- Inline comments go on their own line above the relevant code, not at end-of-line except for very short annotations (e.g., `# noqa`, type ignore directives).
- Keep comments terse and direct. One line is almost always enough. If a comment needs a paragraph, the code needs refactoring.
- Do not write comments that apologize, hedge, or narrate thought process ("this might be wrong", "I think this works", "here we do X").

## Architecture

- Package code lives in `src/bpetite/`. Public imports exposed from `src/bpetite/__init__.py`.
- Internal modules use underscore prefix (`_trainer.py`, `_encoder.py`, `_persistence.py`, `_cli.py`).
- The only public export is `Tokenizer`. Internal modules are not part of the public API.
- `pyproject.toml` is the single source for package and tool configuration.
- Core library has one runtime dependency beyond stdlib: `regex`. No C extensions, no Rust, no external tokenizer libraries.

## Testing

- Framework: pytest. Test files live in `tests/` as `test_<module>.py`.
- No `tests/__init__.py`. Use `importlib` import mode in pytest config.
- Use `@pytest.mark.parametrize` for multi-input coverage — do not duplicate test functions for variants.
- Tests import from the installed `src/` package. Never mutate `PYTHONPATH`.
- Fixtures over setup/teardown. Keep fixtures minimal and explicit.
- All public methods and all critical internal logic must have tests.

## Environment

- `uv` is the only package manager. All commands run via `uv run`.
- Lockfile is committed. Do not regenerate it without asking.
- Python 3.12 is the target interpreter.
- Dev dependencies: `pytest`, `ruff`, `mypy`. Optional: `tiktoken` (comparison only).
- Supported platforms: macOS and Linux. No Windows support in v1.
